What is competitive programming?

Simply put, competitive programming is about solving mathematical / computer science puzzles as fast as possible.

You can enjoy competitive programming in many ways:

  • Enjoying puzzles themselves, coming up with elegant solutions
  • Competing against others in contests
  • In your own way!
  • (Getting a job :///)

Tennis

A and B played a game of tennis. You are given a full sequence of scores achieved by either player, for example, AABBBABABAAAB. Can you determine who won the game?

A typical CP problem

A CP problem has three parts:

  1. Problem statement
  2. Input / Output format
  3. Constraints
                       -------------
    Input ABABB..A --> | black box | --> Output "A won!"
                       -------------

Solution to "Tennis"

Trivially, we can implement the rules of tennis scoring and simulate the game point by point. With persistence, we can get this right.

Think twice, code once. - Waseem Latif

It is sufficient to notice the player who wins the last point must be the winner of the game!

Elegance

In essence, we aim to find a solution that reveals a deep insight into the problem. In most cases, this means the solution is:

  1. Simple
  2. Minimal
  3. Efficient

Making change

Greedy: always make decisions that benefits you the most.

For example, consider the problem of making change with coins. Suppose you need to make £1.65.

1.65=1.00+0.50+0.10+0.051.65 = 1.00 + 0.50 + 0.10 + 0.051.65=1.00+0.50+0.10+0.05

This is how a greedy mindset would work, but whether mindset will always work is another story.

Take a look at https://cses.fi/problemset/task/1090

What would be the best decision when we are assigning gondolas?

Ferris Wheel

Idea: choose a child, and pair them up with the heaviest child such that their total weight does not exceed xxx. Otherwise, send them to a gondola by themself.

Code plan:

  1. Pick the first child in our array
  2. Loop through our array and find the matching child
  3. Delete first / both from the array
  4. Repeat until the array is empty

Ferris Wheel

Idea 2: Since at each step we want to find the heaviest matching child, a good idea is to sort everything first

This way when we are trying find the matching child, we can simply iterate from the back of the array, and stop when we find a match (since we know the children ahead are even lighter)

Idea 3: reuse where the previous index stopped, and start from there.

Book Search

Consider you want to look for a specific book ("Competitive Programming 4") in a bookstore. The best way to do this may be simply to search all the books.

Luckily, the books in the bookstore are sorted in alphabetical order. Then, a reasonable strategy would be look for the area with books that start with "C", and then look for the desired book.

Binary search

Consider this problem, given an array of sorted numbers, determine whether a given number is within the array.

We can generalise our previous idea:

  1. Let's work in binary, due to its simplicity
  2. Abstract all numbers into either 0 or 1.

Abstraction

Change all numbers to 000 if they are <x\lt x<x, otherwise 111. Then the problem becomes: find the first 111 in this array, then check whether its corresponding number is xxx.

Binary search

  1. Maintain left and right pointers
  2. Check the middle element
  3. Half the range as appropriate

Tips to get good

  • PRACTICE: Trains your intuition, leading you to the right idea faster "this problem smells like binary search"

  • LEARN: Study standard algorithms and data structures. A good resource is CP-Algorithms

  • SOLUTION: Read solutions, even for problems you solved. It may uncover a "deeper understanding" of the problem, elegance!

  • HAVE FUN: With enjoyment comes motivation, and with motivation comes consistency and ultimately success!
